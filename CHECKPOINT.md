# CHECKPOINT

## 系统概览

OmniWeave is a local-first, MCP-native code-analysis graph for coding agents. It indexes source files with tree-sitter/WASM, stores symbols and typed edges in SQLite, then exposes structural retrieval tools (`explore`, `node`, `search`, `callers`, `impact`) to agents and the CLI.

## 模块结构

| 模块 | 职责 | 关键公共接口 | 状态 |
|---|---|---|---|
| `src/extraction/` | Parse files, create symbol/file/import nodes, emit unresolved references. | `TreeSitterExtractor`, language extractors, `initGrammars` | Hot path; keep memory/time bounded. |
| `src/resolution/` | Resolve names/imports/framework conventions and synthesize cross-boundary edges. | `resolveReferences`, framework resolvers, `synthesizeCallbackEdges` | Highest complexity area; add edges conservatively with provenance. |
| `src/db/` | SQLite schema, migrations, prepared graph queries, FTS. | `DatabaseConnection`, `QueryBuilder` | Source runs need Node 22.5+ for `node:sqlite`; release bundles vendor Node. |
| `src/mcp/` | MCP transport, daemon/proxy/session lifecycle, tool handlers and output shaping. | `ToolHandler`, `runMcpServer`, `getStaticTools` | Agent-facing quality lives here; output must be sufficient enough to avoid fallback reads. |
| `src/context/` | Format graph context and source snippets under output budgets. | `buildContext`, formatter helpers | Must preserve complete methods or honest omissions, never half bodies. |
| `src/scip/` | Decode existing `index.scip` artifacts and import optional same-language precision facts. | `decodeScipIndexFile`, `importScipIndex` | Optional supplement only; no indexer runner and no core install dependency. |
| `src/installer/` | Install/uninstall agent integrations and steering snippets. | `install`, target adapters | Never overwrite user config without preserving sibling data. |
| `scripts/` | Release, bundle, language-addition, and agent-eval tooling. | `build-bundle.sh`, `pack-npm.sh`, `agent-eval/*` | Release scripts are part of the product contract. |
| `site/` | Astro/Starlight documentation and landing page. | `npm run build` | Static docs site; keep dependency audit clean. |

## 领域词汇表

| 术语 | 含义 | 用于 |
|---|---|---|
| Node | A code symbol, file, route, workflow step, artifact, or external tool. | Graph storage and all tool output. |
| Edge | A typed relationship between two nodes, such as `calls`, `contains`, `imports`, `crossLang`, `produces`, `consumes`, or `invokes`. | Traversal, callers/callees, impact, explore flow. |
| Provenance | How an edge was produced: parsed, resolved, or synthesized. | Trust and honesty boundary. |
| Confidence | Heuristic edge certainty when static proof is incomplete. | Agent-facing caveats and false-positive control. |
| CrossLang | A process/language boundary hop from code or workflow step to a local script file. | OmniWeave's core LSP-gap differentiator. |
| SCIP Fact | A third-party `index.scip` definition/reference/relationship imported with `provenance=scip`. | Optional same-language precision supplement; never a runtime/cross-boundary claim. |
| Shape tax | Fixed overhead from MCP/tool exposure/startup and output form. | Agent A/B interpretation and product strategy. |

## 命名约定

- TypeScript uses ESM, strict types, `async`/`await`, and no `var`.
- Runtime-facing comments explain why a boundary exists, not what obvious code does.
- Agent-facing docs use `OmniWeave`; upstream `codegraph` names appear only in historical acknowledgement or upstream reference docs.
- Release package version, `CHANGELOG.md`, GitHub tag, and npm package version must match.

## 数据流

```
source files
  -> extraction (tree-sitter/WASM + framework file extractors)
  -> SQLite graph (nodes, edges, unresolved_refs, files, FTS)
  -> resolution (imports, name matching, framework resolvers, synthesized edges)
  -> query surfaces (library API, CLI, MCP tools)
  -> agent answer/edit workflow
```

Boundary rules:

- Extraction may skip what cannot be represented honestly.
- Resolution must prefer no edge over a plausible but wrong edge.
- Synthesized edges need `provenance` and metadata such as `synthesizedBy`, `registeredAt`, and confidence where relevant.
- MCP output must be complete enough to use directly, or it must say exactly what was omitted.

## 错误处理模式

- Expected recoverable states return success-shaped guidance: unindexed project, symbol not found, file not indexed.
- Hard failures are reserved for security refusals or real malfunctions.
- Telemetry must fail silent and write only to stderr for notices.
- File access must validate real paths under the project root.

## 设计决策

| 决策 | 选项 | 结论 | 理由 |
|---|---|---|---|
| Product moat | More edges vs lower shape tax | Optimize for bounded structural wins and lower overhead. | Agent A/B shows correctness often ties grep; effort and stability are the actual value. |
| Unknown static behavior | Guess vs skip | Skip or surface a dynamic break. | Wrong edges are worse than missing edges for agent trust. |
| Published npm package | Source package vs generated shim + platform bundles | Release pipeline publishes generated shim packages. | Root package is private development/source package with a `prepublishOnly` guard; `scripts/pack-npm.sh` creates the distribution shape. |
| Source Node compatibility | Major-only CLI gate vs exact engine floor | Exact `>=22.5.0` gate. | `node:sqlite` is absent before Node 22.5, so Node 22.0-22.4 must fail before DB open. |
| Agent guidance | Duplicated install snippets vs MCP/server instructions + repo checkpoint | Keep repo guidance concise and current. | Stale instructions are more damaging than missing verbosity. |
| Read/Grep redirect hooks | Blanket redirect vs output sufficiency | Do not ship blanket redirects. | A/B showed the hook regressed tool count, turns, and wall-clock; improve `explore`/`node` sufficiency instead. |
| Project config | Broad config surface vs one optional extension map | Keep zero-config default; allow only `omniweave.json.extensions` for custom source extensions. | Non-standard extensions are a real indexing blind spot, but broader config would revive complexity that `.gitignore` and built-ins already handle. |
| Upstream feature intake | Cherry-pick everything vs evidence-gated port | Port only small, testable wins that preserve OmniWeave semantics. | The upstream AI/offload track and large same-file value-reference patch need separate product review; local-first trust and false-positive control come first. |
| Explore ranking scope | Treat all indexed source equally vs down-rank external snapshots | Down-rank `research/*/repos/*` and other low-signal support files for ordinary queries; allow them only when the query explicitly asks for research/snapshot/external evidence. | This repo intentionally stores competitor/source snapshots inside the project. They are evidence, not first-party code, and must not steal `explore`'s source budget from the files an agent will edit. |
| Broad explore tokenization | Treat every lowercase word as a named seed vs preserve only symbol-shaped signal | For wide queries, keep distinctive identifiers plus up to two immediately preceding plain method tokens; drop trailing ordinary prose words unless corroborated. | Agents often ask `dispatch proceed HandleThing ClassA`, which must form a call path, but words like `formatting`, `empty`, or `large repo` must not drag unrelated files into the default source budget. |
| Explore source order | Pure graph/reference score vs executable first screen | When callable named seeds exist, non-named type-only/interface-only files sort behind files with actionable source (`function`/`method`/`class`/`route`/`component`). | Type containers such as `src/types.ts` can be connected to everything, but the agent's first screen should show editable logic unless it explicitly asked for type definitions. |
| Empty explore behavior | Bare `No relevant code found` vs recovery guidance | Empty results remain success-shaped and tell the agent to search exact names, target known files, retry with concrete identifiers, or refresh/sync the index; MCP names MCP tools and CLI names shell commands. | Empty retrieval is a recoverable state, not a tool failure; a terse or wrong-channel dead end teaches agents to abandon OmniWeave or read randomly. |
| Snapshot fallback | Use external snapshots when they are the only hit vs require explicit intent | Do not use `research/*/repos/*` snapshots as the fallback for ordinary missing-symbol queries; allow them only when the query says `research`/`snapshot`/`external`/similar. | A missing current-project symbol should not be answered with competitor/source corpus code that happens to share a split token like `Symbol`. |
| Staleness semantics | Treat stale files as unreadable vs separate source freshness from graph freshness | Source blocks may be current because tools re-read disk, but symbols, edges, and line ranges can be stale; stale output points agents to `omniweave_node <path>` for focused current file reads or `omniweave sync` before trusting relationships. | This keeps OmniWeave as the default code-reading surface while staying honest about index freshness and graph-derived context. |
| CLI/MCP explore parity | Trust shared handler by inspection vs cover the shell surface end to end | Keep CLI `explore` routed through `ToolHandler`, but render recovery continuations for the caller's surface; test both built CLI output and MCP ToolHandler output. | Shell-only subagents and non-MCP harnesses rely on CLI `explore`; parity means shared semantics with channel-accurate next steps, backed by executable evidence. |
| Ambiguous explore | Pick one overload silently vs disclose partial overload selection | When a bare query token matches many callable definitions and no owning class/file context disambiguates it, show a compact ambiguity section with kept definitions and examples of other candidates. | `explore` must stay compact, but a trusted structure layer cannot let an agent mistake a ranked subset for the full overload set. |
| Explore hard ceiling | Slice at raw character limit vs preserve complete source blocks | Reserve suffix space and cut to complete fenced source-block boundaries; if none fit, drop the partial source section and ask for a narrower follow-up explore. Sectioned/skeletonized output is described as complete only for displayed ranges, not whole files. | A structure-control layer must never externalize oversized output, leave unterminated code fences, or imply partial/dropped source is already read. |
| Explore relationships | Dump every non-contains edge vs rank and label supporting facts | Relationship output is explicitly not the call path; structural edges (`calls`, cross-boundary/workflow, overrides/implements) sort before weak `references`/`imports`; non-static provenance and confidence are shown inline. | Agents need graph evidence, but they must not mistake incidental references or heuristic edges for a proven execution path. |
